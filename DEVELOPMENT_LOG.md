# Claude Code와 어떻게 이 사이트를 만들었나

이 문서는 README에는 다 못 담은, Claude Code(AI 코딩 도구)와 실제로 어떤 대화를 주고받으며
사이트를 만들어갔는지에 대한 기록입니다. 코드를 한 줄도 직접 안 쳐도, 대화로 어떻게
하나의 웹사이트가 완성되는지 그 흐름을 보여주고 싶어서 정리했어요.

## 1. 시작 — "이걸 웹사이트로 시연하고 싶어"

ComfyUI에서 이미지를 넣으면 영상으로 바꿔주는 워크플로우 파일(JSON) 하나만 들고 시작했습니다.
"동생이 만든 이걸 보여줄 수 있는 웹사이트를 만들고 싶다, 이런 느낌(스크린샷)이면 좋겠다"고
이야기를 건넸어요.

Claude는 바로 코드를 쓰지 않고, 먼저 질문들을 던졌습니다.

- ComfyUI는 어디서 실행돼요?
- 워크플로우가 API로 호출할 수 있는 형태인가요?
- 사이트는 누가, 어떤 용도로 쓰나요?

이 질문들에 답하면서 "아, 이런 것까지 미리 정해야 하는구나" 싶었던 부분들이 같이 정리됐고,
Claude는 이걸 바탕으로 구현 계획을 짜서 먼저 보여주고, 동의를 구한 다음에야 실제 코드를
작성하기 시작했습니다.

## 2. 첫 번째 막힘 — "어디서 돌고 있는 거지?"

처음엔 "동생 PC에서 ComfyUI를 돌리고, 그걸 인터넷에 잠깐 뚫어서 연결하자"는 방향으로
이야기가 흘렀어요. 그런데 동생이 보낸 링크가 `cloud.comfy.org`였던 걸 발견하면서,
사실 동생의 PC가 아니라 **Comfy Cloud**(클라우드 호스팅 서비스) 위에서 작업이 이루어지고
있었다는 게 드러났습니다. 덕분에 PC를 인터넷에 노출시키는 복잡한 방법은 통째로 필요 없어졌고,
대신 Comfy Cloud의 API를 직접 호출하는 더 단순한 구조로 바뀌었습니다.

## 3. 두 번째 막힘 — "로그인부터 하라고?"

API 키를 받아서 연결했는데, 영상 생성을 요청할 때마다 이런 에러가 떴습니다.

> Unauthorized: Please login first to use this node.

캔버스(브라우저)에서 직접 실행하면 영상이 잘 나오는데, API로 똑같은 요청을 보내면 거부되는
상황이었어요. 한 번에 안 풀려서 여러 단계를 거쳤습니다.

1. xAI(Grok)에서 직접 영상 생성 API 키를 받아서, ComfyUI를 거치지 않고 바로 호출하는
   방법으로 우회해볼까 고민 — 하지만 xAI 계정/구독이 없어서 보류.
2. "그래도 ComfyUI로 풀어야 한다"는 방향을 다시 잡고, Claude가 ComfyUI의 공식 문서와
   GitHub 이슈 게시판까지 검색해서 같은 문제를 겪은 다른 사람들의 사례를 찾았습니다.
3. 원인을 찾았습니다: 이 워크플로우 안의 영상 생성 노드는 브라우저 로그인 정보를 따로
   필요로 하는데, API로 보낼 땐 그 정보가 자동으로 같이 전달되지 않는 구조였어요.
4. 해결법도 공식 문서에 있었습니다 — API 요청에 `api_key_comfy_org`라는 값을 직접
   끼워 넣어주면, 그게 "로그인 정보" 역할을 대신 해준다는 것. 이걸 코드에 반영하자
   바로 정상적으로 영상이 만들어지기 시작했습니다.

이 과정은 단순히 "코드를 짜는" 게 아니라, 에러 메시지 하나를 붙잡고 문서와 커뮤니티를
뒤지며 원인을 추적해가는 과정이었어요. 막힐 때마다 "이건 우리 코드 문제가 아니라
플랫폼 쪽 제약일 수 있다"처럼 원인의 위치를 좁혀가며 진행했습니다.

## 4. 화면 다듬기 — 대화로 디자인하기

기능이 동작한 다음엔 화면을 동화책처럼 보이게 다듬는 과정이 이어졌습니다. 전부 말로
설명하면서 조정했어요.

- "시작하기 버튼 누르면 딱 나오게 해줘, 요정 이미지로 꾸며줘" → 첫 화면(랜딩 페이지)과
  버튼 등장 애니메이션 추가
- "오래된 동화책 같은 종이 질감, 고풍스러운 느낌으로" → 종이 질감 텍스처, 손글씨 같은
  타이핑 효과, 금색 장식 문양 추가
- "글감을 더 길게, 구체적으로" → 야근길에 본 쓰레기통 이야기를 바탕으로 9개의 문장으로
  서사를 새로 씀
- "제목을 더 동화틱하게" → 여러 제목 후보 중에서 "아무도 모르는 밤의 요정"으로 결정
- "파비콘(브라우저 탭 아이콘)을 요정으로" → 탭에 뜨는 작은 아이콘까지 직접 그려서 적용

## 5. 배포 — 누구나 들어올 수 있게

마지막으로 **Vercel**이라는 서비스에 사이트를 올렸습니다. 명령어 몇 줄로 배포가
끝났고, 그 뒤로는 `jkant.vercel.app`이라는 링크만 있으면 누구나 바로 접속해서
사진을 올리고 영상을 받아볼 수 있게 됐습니다.
